# ArcGIS Pro SDK Wiki 文件格式

當文件對象是 ArcGIS Pro SDK 的 ViewModel、DockPane 或 Add-in 類別時，使用此格式。

## 文件開頭

```md
# {ViewModel名稱}
```

開頭用 1 到 2 段說明此 ViewModel 所屬功能、負責的 DockPane/UI 狀態、ArcGIS Pro 圖層互動、地圖事件、GDB/影像/GP/Python 等核心責任，以及此類別不負責的範圍。

## 建議章節

### 1. DockPane 生命週期

說明內容：

| 項目 | 說明重點 |
|---|---|
| DockPane ID | 列出 DockPane 或 DAML ID，說明與 UI 顯示的關係。 |
| 初始化 | 說明 `InitializeAsync` 或建構式中建立哪些狀態、資料集合或服務。 |
| 顯示/隱藏 | 說明 `OnShow`、`OnHidden` 或相關生命週期方法。 |
| 事件訂閱 | 若有 `MapSelectionChangedEvent.Subscribe` 等事件，說明訂閱原因與取消訂閱時機。 |

維護提醒：事件未取消、重複訂閱或在錯誤 thread 操作 ArcGIS Pro SDK API，容易造成 UI 狀態錯亂或記憶體問題。

### 2. 核心資料來源

說明主要 GDB、FeatureClass、Table、影像池、輸入路徑、設定檔或外部服務。

重要路徑使用：

```text
{path}
```

建議表格：

| 資料來源 | 類型 | 用途 | 維護注意 |
|---|---|---|---|

### 3. 主要狀態碼

若程式有狀態碼，必須獨立成章。

| 狀態碼 | 意義 | 使用位置 | 維護注意 |
|---|---|---|---|

若程式碼沒有明確狀態碼，寫明「程式碼中未看到明確狀態碼」。

### 4. GDB 或圖層載入

說明如何選取、載入、檢查、加入地圖。

必須注意：

| 項目 | 說明重點 |
|---|---|
| `MapView.Active` | 是否依賴目前作用中的地圖。 |
| `FeatureLayer` | 載入來源、圖層名稱、與 FeatureClass 的關係。 |
| Definition Query | 查詢條件與維護風險。 |
| Renderer | 符號化欄位、狀態對應與變更風險。 |

### 5. 清單查詢或資料篩選

依 RadioButton、狀態、條件或篩選來源分節說明。

| 篩選條件 | 查詢來源 | 回傳資料 | 維護注意 |
|---|---|---|---|

### 6. 圖層顯示與 Renderer

說明圖層名稱、Definition Query、符號化、欄位依據，以及狀態顯示如何對應資料。

### 7. 地圖選取同步

說明地圖點選如何回填 ViewModel 屬性。

必須交代：

| 項目 | 說明 |
|---|---|
| 選取圖層 | 哪些圖層會被處理。 |
| 主鍵欄位 | `OBJECTID`、`PROJECTID` 或其他主要鍵值。 |
| UI 回填 | 地圖選取後更新哪些 ViewModel 屬性或清單。 |
| 例外情境 | 無選取、選到錯誤圖層、多筆選取時如何處理。 |

### 8. 主要資料結構

對 `ObservableCollection`、`Dictionary`、`List` 等資料結構，用表格說明索引或欄位意義。

| 資料結構 | Key/索引 | Value/內容 | 用途 | 維護注意 |
|---|---|---|---|---|

### 9. 加入/編輯/刪除清單的規則

| 規則 | 說明 | 影響 |
|---|---|---|

| 方法/Command | 觸發來源 | 責任 | 維護注意 |
|---|---|---|---|

### 11. 影像、檔案或 GP 處理流程

若有 Python、Raster、Mosaic Dataset、GP Tool，需分章說明。

| 處理項目 | 輸入 | 輸出 | 工具/方法 | 失敗風險 |
|---|---|---|---|---|

### 12. 狀態更新與成果輸出

說明何時更新狀態、輸出哪些檔案、壓縮或複製哪些成果。

| 更新/輸出 | 觸發時機 | 目標位置 | 維護注意 |
|---|---|---|---|

### 13. 紀錄與進度列

說明 log、進度文字、UI 狀態如何更新。

| 項目 | 說明 |
|---|---|
| Log | 記錄內容、位置與錯誤查找方式。 |
| 進度列 | 如何更新進度與狀態文字。 |
| UI 鎖定 | 長時間任務時如何避免重複執行。 |

### 14. 路徑與檔案

用表格列出重要檔案、資料夾與用途。

| 路徑/檔案 | 用途 | 維護注意 |
|---|---|---|

### 15. 維護風險與建議

條列容易壞掉的地方，例如：

- 事件未取消或重複訂閱。
- GDB schema、FeatureClass 欄位或 Table 欄位變動。
- GP Tool 失敗、Python 環境不一致或參數變動。
- 路徑不存在、影像檔缺漏或輸出權限不足。
- `QueuedWorker` 使用不當造成 thread 問題。
- `MapView.Active` 為空或使用者切換地圖造成狀態不一致。
