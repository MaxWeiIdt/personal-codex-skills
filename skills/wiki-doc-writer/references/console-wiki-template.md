# Console / 批次執行程式 Wiki 文件格式

當文件對象是一般 Console、排程程式或批次執行程式時，使用此格式。

## 文件開頭

```md
# {程式名稱}
```

開頭用 1 到 2 段說明此 Console 程式的批次任務、輸入來源、輸出結果、是否適合排程、是否有 UI，以及不負責的範圍。

## 建議章節

### 1. 功能概要

用表格列出每個主要功能與說明。

| 功能 | 說明 | 維護注意 |
|---|---|---|

### 2. 使用情境

說明此程式何時執行、由誰執行、每次執行會完成哪些事情。

可用編號流程：

```text
1. 排程或人工啟動程式
2. 讀取設定
3. 查詢資料來源
4. 轉換或比對資料
5. 寫入結果
6. 輸出 log 或摘要
```

### 3. 主要類別與方法

用表格列出 `Program.Main`、helper method、model class 的責任。

| 類別/方法 | 責任 | 輸入 | 輸出 | 維護注意 |
|---|---|---|---|---|

### 4. 輸入與輸出

分成「輸入設定」、「API 查詢條件」、「輸出結果」。

#### 4.1 輸入設定

`appsettings.json` 設定鍵必須用表格說明。

| 設定鍵 | 用途 | 範例/格式 | 維護注意 |
|---|---|---|---|

#### 4.2 API 查詢條件

API query、SQL where、request parameters 使用 `sql` 或表格呈現。

```sql
-- query / where 條件
```

| 參數 | 用途 | 維護注意 |
|---|---|---|

#### 4.3 輸出結果

| 輸出 | 位置 | 格式 | 維護注意 |
|---|---|---|---|

### 5. 主要處理流程

用編號列出完整執行順序，包含初始化、讀設定、建立暫存資料、查詢 API、資料轉換、空間比對、統計、回寫、寫 log。

### 6. 判斷規則或商業邏輯

將時間窗、文字正規化、空間查詢、統計公式、狀態判定獨立說明。

公式可使用：

```csharp
// formula or condition
```

建議表格：

| 規則 | 判斷條件 | 結果 | 維護注意 |
|---|---|---|---|

### 7. 外部相依與資料來源

列出 ArcGIS SDK、CoreHost、Python 腳本、REST API、NLog、JSON 套件、GDB、FeatureClass 等。

| 相依項目 | 類型 | 用途 | 維護注意 |
|---|---|---|---|

### 8. 錯誤處理與例外情境

用表格列出錯誤情境與處理方式。

必須視程式內容涵蓋：

| 錯誤情境 | 處理方式 | 維護注意 |
|---|---|---|
| API 空回傳 | 說明是否略過、重試或記錄 log。 | 避免誤判為成功。 |
| Token 失效 | 說明錯誤訊息與更新方式。 | Token 生命週期需定期確認。 |
| Python exit code 非 0 | 說明如何中止或記錄錯誤。 | 保留 stderr 方便追查。 |
| schema 缺欄位 | 說明是否中止。 | 欄位異動需同步文件與程式。 |
| 資料筆數為 0 | 說明是否為正常情境。 | 排程監控需分辨無資料與失敗。 |
| 寫入失敗 | 說明交易、重試或 log。 | 避免部分資料寫入造成狀態不一致。 |

### 9. 完整流程摘要

用 `text` 區塊列出從 `Program.Main` 到結束的流程樹。

```text
Program.Main
|-- 讀取設定
|-- 初始化服務或暫存資料
|-- 查詢外部資料
|-- 資料轉換/比對/統計
|-- 寫入 MSSQL 或輸出檔案
|-- 寫入 log
|-- 結束
```

### 10. 路徑與檔案

用表格列出 `.cs`、`appsettings.json`、`NLog.config`、Python 腳本、GDB、logs、model 檔案等用途。

| 路徑/檔案 | 用途 | 維護注意 |
|---|---|---|

### 11. 維護建議

條列 Token、schema、查詢時間窗、空間比對、日誌摘要、排程風險等注意事項。

建議涵蓋：

- Token 或 API 憑證是否會過期。
- `appsettings.json` 查詢時間窗是否符合排程頻率。
- MSSQL schema、API 回傳欄位或 GDB schema 異動時需要同步調整。
- 空間比對規則是否依賴座標系、容差或特定圖層。
- NLog 或 log 檔是否足以判斷排程成功、失敗或無資料。
- 排程重跑時是否會重複寫入或覆蓋結果。
