# Python / 腳本 / 資料處理專案 Wiki 文件格式

當文件對象是 Python 腳本、資料處理程式、ETL pipeline、排程 Python、ArcPy 腳本、API 查詢腳本、批次轉檔或自動化工具時，使用此格式。

## 文件開頭

```md
# {程式或專案名稱}
```

開頭用 1 到 2 段說明此 Python 程式的定位、主要任務、輸入來源、輸出結果、是否適合排程，以及不負責的範圍。

## 建議章節

### 1. 功能概要

用表格列出主要功能、責任與維護注意事項。

| 功能 | 說明 | 維護注意 |
|---|---|---|

### 2. 使用情境

說明此 Python 程式何時執行、由誰或哪個排程觸發、每次執行完成哪些事情。

可用流程摘要：

```text
1. 人工、排程或其他程式啟動 Python
2. 讀取參數與設定
3. 載入輸入資料
4. 清理、轉換、比對或分析資料
5. 輸出檔案、寫入資料庫或呼叫 API
6. 寫入 log 與執行摘要
```

### 3. 執行入口與主要模組

說明 `main()`、CLI entry point、主要 `.py` 檔、class、function 的責任。

| 檔案/函式 | 責任 | 輸入 | 輸出 | 維護注意 |
|---|---|---|---|---|

若有 `argparse`、`click`、`typer` 或環境變數，需說明啟動參數與預設值。

```powershell
python main.py --input data/input.csv --output output/result.csv
```

| 虛擬環境 | `venv`、`conda` | 排程執行時需確認使用正確環境。 |

### 9. Log 與錯誤處理

說明 log 位置、log level、錯誤是否中止、是否重試、是否輸出失敗清單。

| 錯誤情境 | 處理方式 | 維護注意 |
|---|---|---|
| 缺少輸入檔 | 說明是否中止與錯誤訊息。 | 排程需能辨識缺檔。 |
| schema 缺欄位 | 說明是否中止或略過。 | 欄位異動需同步程式與文件。 |
| API 失敗 | 說明重試、timeout 與錯誤紀錄。 | 避免 HTTP 200 但 body 為錯誤時誤判成功。 |
| 套件或環境錯誤 | 說明如何確認 Python executable 與 dependency。 | 排程環境常與開發環境不同。 |
| 輸出寫入失敗 | 說明是否清理暫存檔或回復狀態。 | 避免半成品被下游流程使用。 |

### 12. 維護建議

條列 schema、套件版本、排程帳號、檔案編碼、資料量、API 限流、重跑風險等注意事項。

建議涵蓋：

- `requirements.txt`、`pyproject.toml` 或 `environment.yml` 是否固定版本。
- 排程使用的 Python executable 是否與開發環境一致。
- 輸入檔欄位、編碼、日期格式或座標系是否可能變動。
- 大量資料處理是否需要分批、暫存或記憶體控制。
- API token、連線字串或憑證是否會過期。
- 重跑時是否會覆蓋輸出、重複寫入或造成狀態不一致。
- log 是否足以判斷成功、失敗、無資料與部分成功。
